# 听写跟读功能实现说明

## 目标

实现「选教材/单元 → 单词列表点读中英 → 中文听写（纸笔默写）→ 英文跟读」，支持语速、重复遍数、词间隔、顺序/打乱。纯浏览器，无后端。

## 技术选型

| 项 | 选择 | 原因 |
|----|------|------|
| 框架 | Vite + React + TypeScript | 轻量、本地静态部署 |
| TTS | Web Speech API `speechSynthesis` | 零依赖、中英分 voice、可调 rate |
| 持久化 | localStorage | 设置与导入教材刷新不丢 |

未采用 TypeWords / Qwerty Learner 整仓 fork：主流程是打字背词，与「中文听写流水线」不贴合；GPL 体量大。仅参考其词库章节思路。

## 核心模块

### 1. `src/lib/tts.ts`

- `speak(text, { lang, rate })`：取消当前 utterance 后朗读，onend/onerror 结束 Promise
- `cancelSpeak()`：统一取消，避免叠音
- `ensureVoicesLoaded` / `hasVoiceFor`：检测中英文 voice，缺失时 UI 警告

### 2. `src/lib/queue.ts` — `WordPlayer`

状态机：

```
start(words, mode) → buildQueue(shuffle?)
  for each word:
    dictation: speak(zh) × repeatZh
    shadow:    speak(en) × repeatEn
    wait(gapMs)  // 最后一词后不等
  speak("本单元结束")
  status = done
```

控制：`pause`（cancel + 清 gap）、`resume`、`skip`（索引+1）、`stop`。

### 3. `src/lib/import.ts`

- JSON：校验 id/title/units/words
- CSV：表头 unit,en,zh（中英别名），按 unit 聚合

### 4. `src/App.tsx`

- 教材选择、单元多选、三模式、设置滑块、单词表双按钮
- 听写时可选隐藏英文（纸笔默写）

## 手机网页适配（2026-07-16）

- **手机优先**：默认单列、最大宽约 720px，触控高度 ≥44px  
- 输入框 `font-size: 16px`，避免 iOS 聚焦放大页面  
- 单词由表格改为**卡片**，「英」「中」大按钮  
- 播放中**底部固定控制条**（安全区 `safe-area-inset`），便于单手暂停/下一词  
- 「语速/遍数」折叠面板，减少首屏占用  
- 练习区置顶，先点听写/跟读再滚选教材  

手机访问：同一局域网下用电脑 `npm run dev -- --host`，手机打开显示的 Network 地址。

## 使用方式

```bash
npm install
npm run dev
# 手机访问需暴露局域网：
npm run dev -- --host
```

浏览器打开终端提示的本地地址（默认 `http://localhost:5173`）。

1. 选择「示例教材」或导入 JSON/CSV  
2. 勾选单元  
3. **点读**：列表点「英」「中」  
4. **中文听写**：设语速/中文遍数/间隔/乱序 → 点「中文听写」→ 纸笔默写  
5. **英文跟读**：点「英文跟读」  

## 已知限制与坑

| 问题 | 说明 / 处理 |
|------|-------------|
| Windows 无中文语音 | 设置 → 时间和语言 → 语音 → 添加中文语音包 |
| 音质一般 | 系统 TTS 限制；后续可接 Edge TTS 等（非首版） |
| Chrome 需用户手势才播 | 首次点击「听写/跟读/按钮」后由用户触发 |
| React StrictMode 双挂载 | 开发模式可能重复创建 player；stop 在 cleanup 中处理 |
| Excel CSV 乱码 | 请另存为 **CSV UTF-8** |

## 实现中的注意点

1. `skip` 时必须 `cancelSpeak` + 清 gap 定时器，并用 `forceNext` 标记，由 loop 统一 `index++`，避免「跳过 + 循环自增」双跳。  
2. `rate` 限制在 0.5–1.5，避免极端值导致引擎异常。  
3. 切换教材时停止播放，避免队列指向旧词表。
